# Global Codex CLI Instructions

このファイルは Claude Code (Opus) から委譲されたタスクを実行する際のグローバル指針です。

## 実行モデル

あなたは Claude Code (Opus) からタスクを委譲されて呼び出されています。
プロンプトに記載されたタスクを正確に実行し、スコープ外の変更を行わないでください。

## 禁止事項

- **git commit / git push / git tag を実行しない** — コミット判断は委譲元に任せる
- **git checkout / git switch でブランチを切り替えない** — 現在のブランチで作業する
- **ファイルの削除は、プロンプトで明示的に指示された場合のみ**
- **新規ファイルの作成は、タスクに必要な場合のみ** — 不要なヘルパーや抽象化を追加しない
- **機密情報（.env, credentials, secrets）を読み取ったり出力に含めない**

## コーディング規約

### 共通
- エラーハンドリングを省略しない
- 既存コードのスタイルに合わせる（インデント、命名規則、import 順序）
- コメントは既存コードと同程度にとどめる — 過剰なコメントを追加しない
- 型安全性を維持する（any 禁止、適切な型定義）

### Go
- エラーは `fmt.Errorf("context: %w", err)` でラップ
- テストは `testify/assert` を使用
- `golangci-lint` が通るコードを書く

### TypeScript / React
- strict モード前提（any 禁止）
- React は関数コンポーネント + hooks
- ESLint + Prettier 準拠

## 出力品質

- **実装が完了したら、可能な限りテストを実行して確認する**（`go test`, `npm test`, `pytest` 等）
- lint チェックも実行する（`golangci-lint run`, `npm run lint` 等）
- テストや lint が失敗した場合、修正を試みる

## タスクスコープ

- プロンプトで指示されたタスクのみを実行する
- 「ついでに」の改善、リファクタ、ドキュメント追加は行わない
- 不明点がある場合、推測で進めるのではなく、最も保守的な解釈で実装する
