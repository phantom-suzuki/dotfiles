# ADR Review Prompt (L2 External Reviewer)

`/review-adr` スキルが L2 レビュアー（Codex / Gemini）に渡す共通プロンプトテンプレート。stdin で「このテンプレート + 対象 ADR 本文 + 既存 ADR インデックス」をまとめて投入する。

---

あなたは経験豊富なソフトウェアアーキテクトです。これから提示する **ADR (Architecture Decision Record)** を以下 5 軸でレビューしてください。

## レビュー観点

| 軸コード | 軸名 | 評価対象 |
|---|---|---|
| L | 論理一貫性 | コンテキスト → 判断論点 → 検討選択肢 → 決定 → 理由 → 影響 の鎖が崩れていないか |
| C | 既存 ADR との矛盾 | 参照 ADR の方針との整合、上書き / 非推奨化の明示性、参照番号の実在性 |
| G | 抜け漏れ | スコープ外の明示、検討選択肢 ≥ 2、ネガティブ影響の記載、関連 Issue / ADR リンク |
| A | 軸選定の妥当性 | 中心となる比較軸（投資レベル / 階層モデル等）の選定理由、軸の網羅性 / 独立性 / 代替軸の検討 |
| N | 表記・整合（軽量） | ADR 番号整合、リンク切れ、用語ゆれ。**初出のみ指摘**（CodeRabbit と重複しがちなため） |

## severity 判定

- **must-fix**: 論理矛盾 / 既存 ADR との整合崩れ / 必須セクション欠落
- **should-fix**: 抜け漏れ / 軸選定根拠の弱さ / 検討選択肢の不足
- **nit**: 表記ゆれ / 軽微な抜け漏れ / 改善提案

## 出力形式（厳守）

以下の JSON スキーマで出力してください。**コードブロックは使わず、純粋な JSON のみ**を返してください。

```json
{
  "findings": [
    {
      "axis": "L",
      "category": "must-fix",
      "title": "論点 X が決定セクションで言及されていない",
      "description": "コンテキストで論点 X を提起しているが、決定 / 理由のいずれにも対応する記述がない。",
      "suggestion": "決定セクションに論点 X への対応を追記するか、コンテキスト側で論点を絞ることを推奨。"
    }
  ],
  "overall_judgment": "comment",
  "overall_summary": "全体として論理は通っているが、論点 X の扱いと既存 ADR-099 との整合に修正が必要。"
}
```

`overall_judgment` は `approve` / `comment` / `request-changes` のいずれかを使用してください（peer-review と互換）。

## 注意事項

- **指摘は具体的に**: 該当セクション名（## の見出し）を `description` に含める
- **修正案を必ず提示**: `suggestion` は「何をどう変えれば改善するか」を具体的に書く
- **過剰指摘を避ける**: nit は本当に意味があるもののみ。typo / 細かい言い回しは無視
- **既存 ADR インデックスとの突合**: 提示される既存 ADR インデックスに記載のない ADR 番号への参照があれば C 軸で指摘
- **ADR の合意性を尊重**: 「決定」を覆す指摘ではなく、決定の根拠強化や論理の補強を目的とする
- **日本語で出力**: タイトル・description・suggestion は日本語

---

以降に、対象 ADR の本文と既存 ADR インデックスが続きます。
