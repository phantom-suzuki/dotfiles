# レビュー・改善の観点

1. **一般的に使用しない表現**: 業界・社内で通じない独特な言い回しを、普通の日本語に直す
2. **難しい日本語の言い回し**: 以下の規律で書き直す
   - 一文を短くする（複数の論点を 1 文に詰めない。「〜であり、〜のため、〜だが」の連結を分割）
   - 主語・動作主を明示する（誰が何をするか）
   - 二重否定を使わない
   - 動作名詞の連結（「XX 整備」「XX 把握」「XX 通読」等）を動詞句に開く。ただし見出し・表のセル内の短い名詞句は対象外
   - あいまいな相対表現（「順次」「なるべく」等）は具体値・具体条件に置き換えられるなら置き換える。原文に具体値がない場合は捏造しない
3. **変な造語・略語・記号参照**: 以下を機械的に適用する（初出に説明を添えるだけでは不十分）
   - 工数単位は人日・人月で書く（PD / PM 等の英略語は使わない。既出なら全置換し、参照元が PD 表記なら凡例で注記する）
   - 工程・職能の独自略称（BD / DD / SI / FI 等）は「基本設計 / 詳細設計 / サーバー実装 / フロント実装」等へ展開する。FE・BE・Infra・Native のような一般に通じる職能略称は使ってよい
   - 文書内 ID（M1 / CP-1 / G1 / W1 等）は、定義表・凡例以外では名称を併記する（例: `CP-2（MVP 第一回段階判断）`）
   - 節番号参照（§N.M）は節タイトルを併記する（例: `実現可能性の評価（§2.4）`）
   - 独自のグルーピング語（「必須13」「フル20」等）は凡例で定義・出典明記したうえで使い、可能なら Milestone / Epic / Issue など GitHub 上の単位・粒度で言い換える
   - 外部文書が定義する識別子（{{外部定義の識別子を列挙。例: BD-BE-4 / BE-S{step}}}）は変更せず、初出で役割を併記する
4. **図の挿入**: 文章や表だけでは構造・時系列・フローが把握しにくい節に Mermaid 図を挿入する
   - {{図の現状（0 件なら候補節を指示。既存図がある場合は「既存図はいじらない」と明記）}}
   - 表で十分に伝わる箇所に冗長な図を足さない
5. **ドキュメント整理**: 節の重複・冗長な繰り返し・読み順の混乱があれば整理する。ただし節構造の大改編はしない（後述の制約参照）

# Mermaid 記述規約（必須）

- `classDef` でノード色を指定する場合、`color` を必ず明示する（GitHub ダークモードで文字が埋もれるため）
- 推奨色: Done=`fill:#1a7f37,stroke:#1a7f37,color:#ffffff` / In Progress=`fill:#bf8700,stroke:#9a6700,color:#ffffff` / Warning=`fill:#cf222e,stroke:#82071e,color:#ffffff` / Info=`fill:#0969da,stroke:#0969da,color:#ffffff`。パステル色は禁止
- 色を使う図の直前に凡例を 1 行併記する（例: `> カラールール: 🟢 完了 / 🟡 進行中 / 🔴 警告`）
- ノード ID は ASCII、ラベルは日本語（`id[日本語ラベル]` 形式）。ノード ID とラベルの記号（案 A/B/D 等）を食い違わせない。`stateDiagram-v2` + 日本語 ID はレキサーエラーになるので使わず、`graph TD` / `graph LR` / `flowchart` を使う
- {{既存文書に Mermaid 図がある場合、そのスタイルと整合させる旨を明記}}

# 変更してはならないもの（最重要制約）

- **節番号・見出しテキスト**: {{相互参照の状況を明記。例:「他文書から 20 箇所超の相互参照があるため、見出しの変更は原則禁止」}}
- **数値・日付・件数**: 一切変更しない（数値の捏造・丸めは厳禁）
- {{文書固有のタグ運用。例: **[既定] / [提案] / [要判断] タグ**: 位置・値とも変更しない}}
- **相対リンク・Issue 参照（#NNNN 等）**: パス・番号を変更しない
- **事実関係・判断内容**: 表現だけを直す。意味・結論・推奨を変えない

# その他の制約

- 変更は表現改善に必要な最小限にとどめる。文書全体のトーン（{{である調 / ですます調}}）を維持する
- Markdown のフェンスコードブロックには言語タグを付ける
- git commit / git push は実行しないこと。ファイル編集のみ
- 「疎通」→「痎通」のような中国語字形の漢字を混入させない

# 完了条件

- 対象ファイルの編集が完了している
- 最後に以下を報告する:
  1. 変更箇所の一覧（節番号 + 変更理由の 1 行サマリ、カテゴリ別: 表現修正 / 略語説明追加 / 図追加 / 整理）
  2. 挿入した Mermaid 図の一覧（節番号と図の種類）
  3. 「問題なし」と判断して変更しなかった観点があればその旨
