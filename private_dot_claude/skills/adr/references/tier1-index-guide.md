# CLAUDE.md への ADR 参照設定ガイド

## 概要

CLAUDE.md に ADR ディレクトリの場所とステータスの扱いルールを記載する。
個別 ADR の要約を毎回追記するのではなく、ディレクトリ参照のみを設定する。
AI は必要に応じて ADR ディレクトリを読みに行く。

## 書式

CLAUDE.md に ADR セクションがない場合、以下を追加することを提案する:

```markdown
## 設計判断

- ADR: `docs/adr/`
- ステータスの扱い: accepted = 従う、proposed = 参考情報、deprecated/superseded = 無視
```

パスはプロジェクトの実際の ADR ディレクトリに合わせる。

## ステータスごとの扱い

| ステータス | AI の扱い |
|-----------|----------|
| accepted | **決定事項として従う**。実装時の制約・前提として参照する |
| proposed | 参考情報として認識する。確定前のため実装の制約にはしない |
| rejected | 「検討したが採用しなかった」記録として参照する |
| deprecated | 無効化された決定。現在の実装には適用しない |
| superseded | 新しい ADR に置き換え済み。supersede 先を参照する |

## 提案のタイミング

- CLAUDE.md に ADR ディレクトリへの参照がない場合のみ提案する（初回のみ）
- 既に参照が記載されていれば何もしない
- CLAUDE.md が存在しないプロジェクトでは作成を強制しない
