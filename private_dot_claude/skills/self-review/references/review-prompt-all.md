## レビュー観点: 全観点（従来モード）

`parallel-by-aspect` 以外のモード（`parallel`, `cascade`）ではこのテンプレートを使用し、1 人のレビュアーに全観点を依頼する。小〜中規模 diff ではこれで十分機能する。

### 観点（網羅的にリストアップすること）

1. **バグ・ロジックエラー**（確実に誤った結果を生む問題）
   - 条件分岐の誤り、境界値 / off-by-one、null / undefined の扱い
2. **セキュリティ脆弱性**
   - インジェクション、認証不備、ハードコードされたシークレット、暗号の誤用
3. **エラーハンドリングの漏れ**
   - 例外の握りつぶし、失敗経路の未処理、リソースリーク
4. **エッジケースの未処理**
   - 空入力、巨大入力、境界条件、並行処理の競合
5. **パフォーマンス問題**
   - N+1 クエリ、不要なループ、無駄なアロケーション
6. **設計上の懸念**
   - 責務の混在、不適切な依存方向、API 一貫性欠如、DRY 違反

### 指摘時の注意

- 各指摘に severity を付与する（critical / warning / info）
- 1 つ見つけて止まらず、全問題を網羅的に報告する
- 「可能性がある」レベルの推測は info 扱い
