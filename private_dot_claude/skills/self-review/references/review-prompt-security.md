## レビュー観点: セキュリティ

このレビューは **セキュリティ脆弱性の検出に特化** する。純粋なロジックバグや設計・アーキ面の懸念は別のレビュアーが担当するため、**このレビューでは触れないこと**（重複を避けるため）。

### 重点項目

1. **インジェクション**
   - SQL インジェクション（文字列結合 / placeholder なし / ORM 生クエリ）
   - コマンドインジェクション（`exec`, `spawn`, `shell=True` で外部入力を結合）
   - LDAP / XPath / NoSQL インジェクション
   - テンプレートインジェクション（SSTI）

2. **認証・認可の不備**
   - 認可チェック漏れ（エンドポイントやハンドラで user / role を検証していない）
   - IDOR（他ユーザーのリソース ID を直接受け取り、所有者チェックなしでアクセス）
   - 権限昇格の経路（管理者フラグの直接更新、JWT の署名検証漏れ）
   - セッション固定、セッション ID 再生成漏れ

3. **シークレット・機密情報の漏洩**
   - ハードコードされた API キー / パスワード / トークン
   - ログ・エラーメッセージへの機密値出力
   - `.env` や credential ファイルのコミット
   - URL クエリパラメータへの秘密情報埋め込み

4. **暗号・ハッシュの誤用**
   - 弱いアルゴリズム（MD5 / SHA-1 のパスワードハッシュ、DES、ECB モード）
   - ハードコードされた IV / salt
   - 乱数源の誤用（`Math.random` をトークン生成に使用等）
   - 鍵長不足、証明書検証の無効化（`verify=False`、`rejectUnauthorized: false`）

5. **入力検証・出力エンコーディング**
   - XSS（DOM 挿入時のエスケープ漏れ、`innerHTML` / `dangerouslySetInnerHTML`）
   - SSRF（ユーザー入力で URL 組み立て、内部アドレスへのアクセス未ブロック）
   - パストラバーサル（`../` を含むパスの受け入れ）
   - Open Redirect（ログイン後リダイレクト URL のドメイン未検証）
   - CRLF インジェクション（ヘッダ組み立て）

6. **クロスサイト / CSRF**
   - CSRF トークンなしの state-changing エンドポイント
   - CORS 設定の過剰な緩和（`Access-Control-Allow-Origin: *` + credentials）
   - SameSite 属性の不適切な設定

7. **デシリアライズ・ファイル取り扱い**
   - 信頼できない入力の `pickle` / `yaml.load` / Java の `readObject`
   - ファイルアップロードの拡張子・MIME 未検証、実行可能ファイル配置
   - ZIP Slip（アーカイブ展開時のパス検証漏れ）

8. **ロギング・エラーハンドリング（セキュリティ観点）**
   - 詳細なスタックトレースがクライアントに漏出
   - 認証失敗の原因を詳細に返す（ユーザー名列挙）
   - レート制限なしの認証エンドポイント

### 指摘時の注意

- 脅威シナリオ（攻撃者は何ができるか）を description に含めること
- 公開されている OWASP / CWE カテゴリがあれば参照する
- 「秘匿チャネルなら安全」などの弱い前提に依存しない

### このレビューで扱わないこと

- 純粋なロジックバグ、null チェック漏れ → bug レビュアーが担当
- 責務分離、API 設計、DRY 違反 → design レビュアーが担当
- 命名、コメント、フォーマット → リンターの領域
