# Task Delegation Rules

セッション消費を最適化するため、タスクの性質に応じて適切なモデル/ツールに委譲する。

## Tier 定義

| Tier | 実行先 | 判断基準 |
|------|--------|---------|
| **T1: 判断層** | Opus（自身） | 設計判断・要件分析・コードベース横断の理解が必要 |
| **T2: 実装層** | Codex CLI | 明確に定義された実装タスク、テスト作成、メカニカルなリファクタ |
| **T3: 補助層** | Sonnet/Haiku サブエージェント | 検索・要約・フォーマット変換・boilerplate 生成 |

## T1: Opus が自ら行うタスク

以下は委譲せず自身で処理する:
- 要件定義・仕様設計・アーキテクチャ判断
- 複数ファイル横断の分析・デバッグ戦略の立案
- コードレビュー方針の決定
- タスク分解とオーケストレーション（委譲先への指示作成）
- 委譲結果の検証と統合判断
- ユーザーとの対話・確認が必要な判断

## T2: Codex CLI に委譲するタスク

### 委譲条件
- タスクの意図と境界が明確に定義できる
- 必要なコンテキストが限定的（対象ファイル + 関連インターフェース程度）
- 設計判断が不要、または Opus が事前に方針を決定済み

### 対象タスク例
- 関数・メソッド・エンドポイントの実装（設計は Opus が決定済み）
- テストコードの作成（テスト対象と方針は Opus が指定）
- メカニカルなリファクタリング（リネーム、import 整理、コールサイト更新）
- 定型ドキュメント生成（API ドキュメント、GoDoc/JSDoc コメント、CHANGELOG）
- lint/format エラーの一括修正
- マイグレーションファイルの生成

### 呼び出しパターン

```bash
# 基本形: full-auto + workspace-write で自律実行
codex exec --full-auto "プロンプト"

# プロジェクトの AGENTS.md を参照させるため、対象ディレクトリで実行する
cd /path/to/project && codex exec --full-auto "プロンプト"
```

### プロンプト構成テンプレート

Codex に渡すプロンプトには以下を含める:

```
## タスク
（何を実装するか）

## コンテキスト
（関連ファイル、インターフェース定義、依存関係）

## 制約
（コーディング規約、使用するライブラリ、避けるべきパターン）

## 完了条件
（テストが通る、lint が通る、特定の動作をする等）
```

### 安全制約
- **Codex にはコミット・プッシュさせない** — 実装のみ行わせ、コミット判断は Opus（ユーザー承認フロー）に残す
- プロンプトに `git commit や git push は実行しないこと` を明記する
- 結果は必ず Opus が diff を確認してから受け入れる

## T3: Sonnet/Haiku サブエージェントに委譲するタスク

### 委譲条件
- Opus ほどの推論力は不要
- Codex に外出しするほどの実装量ではない
- コンテキスト内で完結する補助作業

### 対象タスク例
- コードベースの検索・要約（「この関数の呼び出し元を全部リストして」）
- 既存コードの説明生成
- フォーマット変換（JSON → YAML、テーブル → リスト等）
- boilerplate・scaffold の生成
- エラーメッセージの調査・解釈

### 呼び出しパターン

```
Agent ツールで model: "haiku" または model: "sonnet" を指定
```

## 委譲の判断フロー

```
ユーザーからのリクエスト
  ↓
1. タスクの性質を分析
  ↓
2. 設計判断が必要？ → Yes → T1（Opus が自ら実行）
  ↓ No
3. コード実装・テスト・リファクタ？ → Yes → T2（Codex に委譲）
  ↓ No
4. 検索・要約・補助作業？ → Yes → T3（Sonnet/Haiku に委譲）
  ↓ No
5. T1（Opus が自ら実行）
```

## 委譲時の原則

- **委譲前に必ず設計を完了する** — 「何を作るか」を Opus が決め、「どう作るか」を Codex に任せる
- **1回の委譲 = 1つの明確なタスク** — 複数タスクは個別に委譲する
- **結果検証を省略しない** — Codex の出力は必ず diff 確認してから受け入れる
- **コンテキストは必要最小限で渡す** — Codex の 272K 制限を意識し、関連ファイルを絞る
- **失敗時は Opus にフォールバック** — Codex が期待通りの結果を返さない場合、Opus 自身で対応する
